# 八、javascript 高级

## 8.1 面向对象(高内聚低偶)

> 编程思想: 将以往生活经验总结,使用在编程
> 面向过程编程: 需要关注每一个步骤,每一步都要很清楚,注重中间过程步骤
> 面向对象编程: 找对象,让对象来完成需求,注重整体结果

编程中的对象 和 现实生活中的对象

1. 现实中的对象: 可以看得见摸得着的具体事物
   比如: 我现在使用的这台电脑,你手中的那台手机----都是对象
   现实中的对象 可以通过 对象的特征 和 行为 来描述对象
2. 编程中的对象 其实就是现实中的对象抽象出来的
   现实中对象的 特征------抽象------编程对象的属性
   现实中对象的 行为------抽象------编程对象的方法(函数)

### 创建对象的四种方式

#### 字面量创建

```javascript
let obj = {
  name: 'xiaobin',
  age: 18,
  gender: '男',
}
```

#### 内置构造函数创建

```javascript
let obj = new Object()
obj.name = 'xiaobin'
obj.age = 18
obj.gender = '男'
```

#### 工厂函数创建

```javascript
function createObj(name, age, gender) {
  let obj = {}
  obj.name = name
  obj.age = age
  obj.gender = gender
  return obj
}
let obj2 = createObj('xiaobin', 18, '男')
```

#### 自定义构造函数创建

> 构造函数作用: 创建对象
> 我们也将 `构造函数创建对象` 称之为 实例化对象
> 我们也将 `构造函数创建出来的对象` 称之为 实例对象
> 构造函数创建对象的语法: new 构造函数(参数)
> 返回值: 实例对象

1. 构造函数名一般首字母大写
1. 构造函数中的 this 关键字,在实例化对象的时候 指向实例对象,使用自定义的 构造函数创建对象---实例化对象
1. 构造函数通过 new 语法 实例化对象的时候,可以省略小括号,但是不建议省略
1. 构造函数内不需要 return

- new 构造函数(); 默认返回值就是实例对象
- 如果在函数内写了 return,且 return 的是基本数据类型,则写了也是白写
- 如果在函数内写了 return,且 return 的是复杂数据类型,则构造函数白写了

```javascript
function CreateObj(name, age, gender) {
  this.name = name
  this.age = age
  this.gender = gender
}
let obj = new createObj('xiaobin', 18, '男')
console.log(obj)
```

## 8.2 原型

> 在创建函数的时候,浏览器会自动的创建一个对象,这个对象就是原型对象

1. 函数和原型对象的关系
   - 函数有一个属性 prototype,函数的这个属性指向 对应的原型对象
   - 原型对象有一个属性 constructor, 原型对象的这个属性指向 对应的构造函数
2. 实例对象 和 构造函数的关系
   - 实例对象 是 构造函数 通过 new 语法创建的
3. 实例对象 和 原型对象的关系
   - 实例对象有一个属性 **proto**,对象的这个属性 指向 对应的原型对象

```javascript
// 自定义构造函数
function Person(name) {
  this.name = name
}

// 实例化对象
let p1 = new Person('zs')

// 函数的 prototype 属性 指向 原型对象
console.log(Person.prototype) //  原型对象

// 原型对象的 constructor 属性 ===> 指向 构造函数
console.log(Person.prototype.constructor === Person) // true

// 实例对象的__proto__===>指向 创建这个对象的构造函数对应的原型对象
console.log(p1.__proto__ === Person.prototype) // true
```

注意:

1. 对象的**proto**属性,对象的这个属性指向一个原型对象,但是**proto**不是标准属性,所以一般我们要给原型对象添加方法属性,都是通过 构造函数.prototype 的原型对象添加
2. 原型对象中的方法,是对象实例对象用的,实例对象可以直接访问对应的原型对象中的方法属性

## 8.3 原型链

> 对象都有一个属性 **proto**,而且这个属性指向 一个原型对象，原型对象 也是对象,所以原型对象也有**proto**属性,也是指向一个原型对象

```javascript
// 实例对象的__proto__ 指向 构造函数对应的原型
console.log(p1.__proto__ === Person.prototype) // true
console.log(p1.__proto__) // 指向原型对象

// p1.__proto__的原型对象 也有__proto__属性,而且也是指向一个原型对象
// p1.__proto__.__proto__ 指向 Object构造函数对应的原型对象
console.log(p1.__proto__.__proto__ === Object.prototype) // true

// 顶级构造函数 Object 对应的原型对象没有 __proto__属性
// 如果非得去访问 则指向null
console.log(Object.prototype)
console.log(Object.prototype.__proto__) // null
// 原型链: 对象 通过 __proto__一直链接起来的一个链状结构,就是原型链
```

### 1. 原型链的访问规则 (背诵)

实例对象访问一个属性的时候 先在对象本身查找,如果有则使用 如果没有,则去对象**proto**指向的原型对象中查找,如果有则使用 如果还是没有找到,则沿着原型链中的**proto**下一个原型对象中查找,找到了就使用 如果一直找到顶级构造函数对应的原型对象中,还是没有则返回 undefined

### 2. 对象的赋值规则

给对象赋值的时候,是直接在这个对象身上添加属性或修改属性,和原型无关

### 拓展

1. 任何对象的原型链最后 都是指向 Object 构造函数的原型
2. 函数是一个特殊的数据类型----是 js 中的一等公民 函数既可以当做对象使用,也可以当做构造函数使用
3. js 中万物皆对象 甚至基本数据类型中的字符串,数值,布尔值等等都是 包装数据类型

### 面向对象选项卡案列

```javascript
function Tabs(select, type) {
  let ele = document.querySelector(select)
  this.btnBox = ele.querySelectorAll('.btnBox li')
  this.tabBox = ele.querySelectorAll('.tabBox li')
  this.type = type
  this.change()
}
Tabs.prototype.change = function () {
  for (let i = 0; i < this.btnBox.length; i++) {
    this.btnBox[i].addEventListener(this.type, () => {
      for (var j = 0; j < this.btnBox.length; j++) {
        this.btnBox[j].className = ''
        this.tabBox[j].className = ''
      }
      this.btnBox[i].className = 'active'
      this.tabBox[i].className = 'active'
    })
  }
}
```

## 8.4 ES6 类

- ES6 中引入了 类 的概念(特别的构造函数)
- 类的作用和构造函数的作用一样,都是用于实例化对象
- 但是类不能单做普通函数直接调用
- 使用类 实例化对象语法: new 类名

```javascript
类语法:
  class 类名{
    constructor(){
    }
  }
```

注意:

1. 类名和构造函数命名规则一样
2. 类中默认有一个 constructor 方法,我们称之为构造器 而且在实例化对象的时候,默认会执行这个构造器方法 构造器方法中的 this 指向 实例对象 如果要 给实例对象添加属性或方法的话,则在构造器方法中书写
3. 直接书写在类中的方法,相当于是给类对应的原型对象添加方法
4. 类字段: 直接书写在类中的属性,详单与给实例对象添加的属性
5. 静态方法和静态属性: 通过 static 修饰的类中的属性和方法,

注意: 静态方法和静态属性,只能给类使用,实例对象不能使用 相当于 把类当作对象看待,给这个对象添加的属性和方法

```javascript
class CreateObj {
  // 构造函数
  constructor(name) {
    // this 指向实例化对象
    this.name = name
  }
  say() {
    console.log(this.name, 'hello')
  }
}
let obj = new CreateObj('xiaobin')
console.log(obj) // 实例化对象输出
obj.say() // xiaobin hello
```
